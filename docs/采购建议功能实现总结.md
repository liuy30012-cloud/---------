## 🎯 核心功能实现

### 智能推荐算法 ✅

#### 1. 缺货且有预约 - 最高优先级 ✅
- **触发条件**: 可借副本 = 0 且 预约人数 > 0
- **优先级分数**: +100
- **建议副本**: 当前副本 + max(预约人数, 2)
- **提示信息**: "缺货且有X人预约，急需补充库存"

#### 2. 高借阅频率 ✅
- **触发条件**: 借阅次数 ≥ 15
- **优先级分数**: +80
- **建议副本**: 当前副本 + max(借阅次数/10, 2)
- **提示信息**: "热门书籍，借阅X次，建议增加库存"

#### 3. 中等借阅 + 低库存 ✅
- **触发条件**: 借阅次数 ≥ 8 且 可借副本 ≤ 30%
- **优先级分数**: +60
- **建议副本**: 当前副本 + 2
- **提示信息**: "借阅X次且库存不足，建议适当增加"

#### 4. 有预约但未缺货 ✅
- **触发条件**: 预约人数 > 0 且 可借副本 ≤ 2
- **优先级分数**: +50
- **建议副本**: 当前副本 + 预约人数
- **提示信息**: "有X人预约，库存紧张"

#### 5. 一般借阅但缺货 ✅
- **触发条件**: 借阅次数 ≥ 5 且 可借副本 = 0
- **优先级分数**: +40
- **建议副本**: 当前副本 + 1
- **提示信息**: "借阅频率一般但已缺货，建议补充"

### 优先级评分系统 ✅

```java
// 基础分数（根据场景）
score = 基础分数 (40-100)

// 借阅次数加分
score += borrowCount * 2.0

// 预约人数加分
score += reservationCount * 10.0

// 库存紧张程度加分
if (availableCopies == 0) {
    score += 30.0
} else if (availableCopies <= currentCopies * 0.2) {
    score += 15.0
}

// 优先级判定
if (score >= 100.0) priority = "HIGH"
else if (score >= 60.0) priority = "MEDIUM"
else priority = "LOW"
```

## 📊 推荐算法详解

### 场景1: 缺货且有预约（最紧急）
```java
if (availableCopies == 0 && reservationCount > 0) {
    score = 100.0;
    suggestedCopies = currentCopies + max(reservationCount, 2);
    // 例: 当前5本，3人预约 → 建议8本（增加3本）
}
```

### 场景2: 热门书籍（高需求）
```java
if (borrowCount >= 15) {
    score = 80.0;
    suggestedCopies = currentCopies + max(borrowCount/10, 2);
    // 例: 当前5本，借阅25次 → 建议7本（增加2本）
}
```

### 场景3: 中等需求 + 库存不足
```java
if (borrowCount >= 8 && availableCopies <= currentCopies * 0.3) {
    score = 60.0;
    suggestedCopies = currentCopies + 2;
    // 例: 当前5本，借阅10次，可借1本 → 建议7本（增加2本）
}
```

### 综合评分示例
```
书籍A:
- 基础分数: 100 (缺货且有预约)
- 借阅次数: 20次 → +40分
- 预约人数: 3人 → +30分
- 缺货状态: → +30分
- 总分: 200分 → 高优先级

书籍B:
- 基础分数: 60 (中等借阅+低库存)
- 借阅次数: 10次 → +20分
- 预约人数: 0人 → +0分
- 库存紧张: → +15分
- 总分: 95分 → 中优先级
```
