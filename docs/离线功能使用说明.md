# 离线功能使用说明

## 功能概述

图书馆书籍定位系统现已支持离线功能,即使在网络信号较差或完全离线的情况下,用户也能查看已缓存的书籍信息。

## 核心特性

### 1. 自动缓存
- **热门书籍预缓存**: 系统自动缓存借阅量最高的 100 本书籍
- **浏览历史缓存**: 用户查看过的书籍会自动保存到本地
- **智能更新**: 联网时自动后台更新缓存数据

### 2. 离线浏览
- 离线状态下可以搜索和查看已缓存的书籍
- 查看书籍详情、位置信息、简介等基本信息
- 无缝切换,用户几乎感知不到离线状态

### 3. 缓存管理
- 查看缓存统计(已缓存书籍数量、最后更新时间)
- 手动触发缓存更新
- 清空所有缓存数据

## 技术实现

### 架构
- **Service Worker**: PWA 标准方案,拦截网络请求实现缓存策略
- **IndexedDB**: 本地数据库,存储书籍数据和元数据
- **Cache API**: 缓存静态资源(CSS/JS/图片)

### 缓存策略
1. **静态资源**: Cache First (缓存优先)
2. **热门书籍**: Stale-While-Revalidate (后台更新)
3. **用户浏览书籍**: Network First (网络优先)
4. **搜索请求**: Network Only (仅网络)

## 使用方法

### 开发环境启用
在开发环境中,Service Worker 默认不启用。如需测试离线功能,设置环境变量:

```bash
# .env.local
VITE_ENABLE_SW=true
```

### 生产环境
生产环境自动启用离线功能,无需额外配置。

### 使用缓存管理界面
1. 在设置页面中添加缓存管理组件
2. 查看缓存统计信息
3. 点击"立即更新缓存"手动更新
4. 点击"清空所有缓存"清除数据

## 用户体验

### 在线状态
- 正常使用所有功能
- 后台自动更新缓存数据
- 无任何提示

### 离线状态
- 顶部显示"离线模式"提示条
- 可以搜索和查看已缓存的书籍
- 尝试借阅/预约时提示"需要网络连接"
- 搜索功能提示"搜索功能需要网络连接"

### 网络恢复
- 自动检测网络恢复
- 后台静默更新缓存
- 提示"已恢复网络连接"

## 注意事项

1. **首次使用**: 首次访问时需要联网加载资源和缓存数据
2. **存储空间**: IndexedDB 通常有几百 MB 的存储限额,足够缓存大量书籍
3. **数据时效性**: 离线数据可能不是最新的,建议定期联网更新
4. **功能限制**: 借阅、预约、评价等操作必须在线完成

## 后续优化建议

1. **后台同步**: 使用 Background Sync API 实现离线操作队列
2. **推送通知**: 使用 Push API 实现到期提醒等通知
3. **增量更新**: 只更新变化的数据,减少流量消耗
4. **缓存策略优化**: 根据用户使用习惯动态调整缓存策略
5. **存储配额管理**: 监控存储使用情况,自动清理过期数据

3. 查看 Service Workers、Cache Storage、IndexedDB
4. 使用 Network 标签的 Offline 模式模拟离线

### 实际测试
1. 正常浏览几本书籍
2. 断开网络连接
3. 刷新页面,验证离线功能
4. 尝试搜索已缓存的书籍
5. 查看书籍详情
6. 尝试借阅操作(应提示需要网络)

## 故障排查

### Service Worker 未注册
- 检查浏览器是否支持 Service Worker
- 确认是 HTTPS 或 localhost 环境
- 查看控制台是否有错误信息

### 缓存未生效
- 检查 IndexedDB 是否正常初始化
- 查看 Service Worker 是否正常拦截请求
- 确认网络请求是否被正确缓存

### 离线状态未显示
- 检查 OfflineIndicator 组件是否正确集成
- 确认 useNetworkStatus 是否正常工作
- 查看浏览器的 online/offline 事件
