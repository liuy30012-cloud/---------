# 2026-04-22 图书馆首页 HeroSection 视觉重设计

## 背景

当前首页已经具备东方意境基础（山水、雾气、竹影、双时钟），但首屏在产品质感上仍偏“概念展示”，缺少成熟数字产品首页应有的结构秩序与视觉克制。

本次改版的目标不是推翻现有东方气质，而是在保留核心识别元素的前提下，把首页提升为：**更现代、更精致、更像正式产品入口** 的版本。

用户已经明确确认以下方向：
- 保留东方意境，但整体更现代、更精致
- 优先提升整体质感
- 首页采用“均衡双核”思路：标题负责气质，搜索负责转化，两者都要强
- 必须保留：山水 / 雾气 / 竹影、左侧双时钟区
- 动效强度采用偏戏剧化，但戏剧感主要来自景深与层次，而不是夸张文字或喧闹动画
- 明确否定“海报式巨型标题”方向，最终确认采用“雅致标题 + 强搜索主角”

## 设计结论

### 1. 总体设计方向

采用：**现代策展的结构秩序 + 戏剧景深的氛围表达**。

也就是：
- 页面结构上更像成熟产品首页，信息层级清楚、操作目标明确
- 氛围上保留东方山水语汇，并通过景深、雾层、竹影和滚动层次制造开场感
- 戏剧感存在于空间与动效中，不存在于夸张标题中

### 2. 页面结构

最终采用结构方案 **A**：**单主搜索区 + 下方三张价值卡**。

首屏结构如下：

#### 左侧：双时钟信息塔
- 保留当前双时钟设计（地支时钟 + 现代时钟）
- 视觉上收敛为更克制、更精致的信息塔
- 作为首页识别点与东方时间感补充，不再与主标题抢主视觉中心
- 卡片样式统一提升，减少“堆叠组件感”，增强整体性

#### 中部上方：雅致标题与说明
- 标题不再使用海报式超大字号
- 控制在成熟产品首页可接受的视觉尺度内
- 通过字体、字重、留白、行高建立气质，而不是靠字号压迫用户
- 副文案负责解释首页用途：输入条件、进入统一结果页、保留查询状态，强调连续检索和借阅动作效率

#### 中部核心：主搜索面板
- 搜索面板为首页绝对视觉主角
- 搜索框尺寸、内边距、按钮权重、阴影层级都明显高于其他模块
- 搜索框、按钮、自动补全、热门标签在同一视觉系统中整合，形成单一高价值操作区
- 按钮继续保留明确动作导向文案，例如“进入结果页”

#### 中部下方：热门标签
- 作为搜索区的快速入口，保留但不喧宾夺主
- 优先级低于输入框本体与主按钮
- 作用是降低首次操作成本，而不是承担展示职责

#### 底部承接：三张价值卡
- 保留三张价值卡结构
- 改造为更轻、更精致、更短文案的产品价值说明卡
- 作为搜索区的承接信息，而非并列主视觉模块
- 重点表达：结果可信、状态可回放、搜索后可直达后续动作

### 3. 背景与视觉氛围

#### 保留元素
- 山水
- 雾气
- 竹影
- 柔和光晕 / 暖色纸面感氛围

#### 视觉原则
- 背景负责空间层次，不负责信息承载
- 不能因为追求氛围牺牲正文对比度、按钮识别度或输入框可读性
- 气氛应该像“展陈环境”，不是像“插画主画面”

#### 视觉建议
- 山体层级更清楚：远山 / 中景 / 近景分层
- 雾层更柔，不做大面积糊面遮挡
- 竹影保留边缘存在感，用于制造框景与景深
- 高光与暖色纸感用于增强质感，但不做过度金色装饰

### 4. 交互与动效

整体动效气质：**偏戏剧化，但有秩序、有边界**。

#### 入场顺序
1. 左侧信息塔先淡入
2. 标题与说明进入
3. 搜索面板进入
4. 热门标签与底部三张价值卡补入

要求：
- 入场是有节奏的“揭幕感”
- 不能碎、不能乱、不能到处都动

#### 景深滚动
- 远山、雾层、竹影采用不同速度轻位移
- 制造空间深度与舞台开场感
- 正文区域、搜索框、按钮保持稳定，不参与夸张漂移

#### 搜索交互
- 搜索框 focus 后，边框、阴影、按钮状态增强
- 用户一眼能感知“这里是主操作区”
- 自动补全弹层延续整体高质感风格，但优先保证清晰与点击效率

#### 按钮与标签反馈
- 主按钮 hover：轻微抬升、阴影增强、细微光泽变化
- 热门标签 hover：细腻反馈即可，不做夸张弹跳或过度动画

#### 移动端策略
- 缩弱景深效果与背景存在感
- 优先保证标题、搜索框、按钮的清晰度与点按效率
- 时钟信息塔可重新排布，但仍需保留

### 5. 文案与信息表达

#### 标题风格
- 从“夸张视觉口号”转为“成熟产品价值表达”
- 仍保留一定书卷气，但必须清楚、自然、不装饰化
- 标题应该服务理解，不应成为视觉负担

#### 副文案
- 清楚说明首页职责：输入条件 → 进入统一结果页 → 保留查询状态
- 强调结果页是后续浏览、筛选、借阅动作的起点

#### 搜索提示语
- 保持直接，例如“输入书名、作者或 ISBN”
- 不使用文艺化占位文案

#### 按钮文案
- 保留明确的动作导向表达
- 优先选择能体现下一步动作的词，而不是抽象词

#### 三张价值卡文案
- 改短
- 改清楚
- 改成产品价值说明，而不是装饰性 slogan

### 6. 与当前实现的关系

本次设计应优先基于现有首页 HeroSection 的结构与已有动画系统做重构，而不是另起炉灶。

重点复用与调整对象：
- `frontend/src/components/home/HeroSection.vue`
- 现有山水、雾气、竹影 SVG / 背景结构
- 左侧双时钟区现有组件接入方式
- 现有搜索自动补全、标签点击、搜索提交行为
- 现有 GSAP / ScrollTrigger 动画结构

设计目标是：
- 保留已验证有效的交互行为
- 重构视觉层级、空间比例、信息秩序和动效节奏
- 避免做与当前架构相冲突的大范围推倒重来

## 实现边界建议

这次改版聚焦首页 HeroSection 首屏，不扩展到以下范围：
- 不重做结果页
- 不修改搜索业务逻辑
- 不新增复杂首页业务模块
- 不额外引入与首页目标无关的装饰组件

允许的调整包括：
- HeroSection 模板结构重排
- HeroSection 样式全面重构
- 动效节奏与参数重调
- 文案优化
- 三张价值卡与热门标签的视觉层级调整
- 左侧双时钟区容器样式重构

## 验收标准

改版完成后，应满足以下结果：

1. 用户第一眼感受是“更成熟、更高级的产品首页”，而不是“概念展示页”
2. 标题有气质，但不会再出现巨型海报字带来的压迫感与违和感
3. 搜索区成为首屏最清晰的操作核心
4. 东方山水氛围仍然成立，且比当前更有层次
5. 动效具有戏剧性，但不影响可读性与操作效率
6. 左侧双时钟区被保留，并与整体风格协调
7. 三张价值卡存在感降低，但信息表达更有效
8. 移动端仍保持清晰易用

## 需要修改的关键文件

- `frontend/src/components/home/HeroSection.vue`
- 如需同步变量：`frontend/src/styles/variables.css`
- 如需影响全局布局，再审慎评估：`frontend/src/styles/app.css`

## 备注

本设计已经过用户逐段确认，以下结论已明确冻结：
- 不使用巨型大字标题
- 结构采用 A
- 视觉气质采用“保留东方意境，但更现代、更精致”
- 标题与搜索采用“雅致标题 + 强搜索主角”
- 动效采用“偏戏剧化，但戏剧感来自景深与层次”
